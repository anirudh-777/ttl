ttl

v1.0.0

Terminal Task List — a tiny, AI-friendly task tracker.

A single static binary (~13 MB) with CLI, TUI, web UI, REST API, and MCP server. Multi-tenant, SQLite-backed, no CGO.

Install

Download a release binary from your project's dist/ folder, or build from source:

go install github.com/anirudh-777/ttl/cmd/ttl@latest
# or
git clone ttl && cd ttl && make build

Place ttl somewhere on your $PATH. That's it — no daemon, no service to register.

Quick start

Two terminals. One runs the server, the other talks to it.

# 1. Server (terminal A)
ttl serve --addr :8093 --db ~/ttl.db

# 2. Sign up (terminal B) — prints your API key
ttl signup --server http://localhost:8093

# 3. Capture a few tasks
ttl add "Write docs"        --project docs  --tag p1
ttl add "Review PR #42"     --project eng   --due tomorrow
ttl add "Weekly standup"    --repeat 'rrule:FREQ=WEEKLY;BYDAY=MO'

# 4. Review
ttl today          # what's due / in progress
ttl inbox          # everything else
ttl view today     # open the TUI today view

# 5. Complete
ttl done <task-id>                # shows next spawn if recurring

Mental model

ConceptWhat it is
TenantIsolation boundary. Every user belongs to one. Data never crosses tenants.
UserBelongs to a tenant, has bcrypt password + API keys.
ProjectA named bucket for tasks.
TagFree-form label. A task can have many.
TaskThe atom. Title, project, tags, due, recurrence, notes, subtasks.
Time entryA timer run on a task (or on “general”). Records started_at / ended_at.
ReminderA timestamped note. Background ticker fires OS notifications.
IntegrationA linked external issue tracker (GitHub, Linear). Issues mirror as tasks.
Multi-tenancy is enforced at two layers: the type system requires every store call to carry a *tenant.Context, and every SQL query includes tenant_id in its WHERE. There's no global path.

Capture & review

Add

ttl add "title" [--project p] [--tag t] [--due 2026-07-01] [--repeat weekly] [--notes "..."]
ttl subtask add <task-id> "subtask"
ttl add "title" --format json

List & filter

ttl list                               # open tasks
ttl list --project docs                # filter by project
ttl list --view upcoming               # smart view
ttl list -o ndjson | jq -r '.id'       # pipeline

Output formats

Every list command accepts -o text|json|ndjson. NDJSON is one record per line — ideal for agents and jq.

TUI keys

  • j/k or ↑/↓ — move
  • Space — mark done
  • e — edit title
  • d, then y — move to trash
  • n — new task
  • 1–7 — switch smart view
  • q — quit

Focus & time

ttl start <task-id>           # or omit id for "general"
ttl stop
ttl timer                      # what's running

ttl pomodoro <task-id> --minutes 25

ttl log                        # today's worklog summary

Only one timer per user can run at a time. Starting a new one without stopping the old one returns the active timer.

Recurring & reminders

Recurring tasks

ttl add "Standup"  --repeat 'rrule:FREQ=WEEKLY;BYDAY=MO'
ttl add "Pay rent" --repeat monthly
ttl add "Backup"   --repeat daily

Completing a recurring task spawns and prints the next occurrence:

ttl done <task-id>
# task abc closed
# next occurrence: def (due 2026-07-04)

Reminders

ttl reminder add <task-id> --at +30m
ttl reminder list
ttl reminder delete <id>

The server checks for due reminders every 60 seconds and fires a best-effort OS notification (osascript, notify-send, or msg).

AI agents (MCP)

ttl mcp speaks JSON-RPC 2.0 over stdio with structured tools for tasks, projects, reminders, timers, and workspace administration:

ToolWhat it does
add_taskCreate a task
list_tasksFilter and list
show_taskFetch by id
search_tasksFree-text + tag search
complete_taskMark done (spawns next if recurring)
delete_taskDelete by id
start_timerBegin timer on a task
stop_timerEnd current timer
active_timerWhat's running now
worklog_todayToday's worklog summary

Wire it into Claude Desktop / Cursor / Cline as a stdio MCP server. See docs/mcp.md in the source repo for the JSON config.

Integrations

ttl integrations add github --label "my-work"
ttl integrations add linear --label "my-team"
ttl integrations list
ttl integrations sync <id>
ttl integrations remove <id>

For instant updates, set the webhook URL the installer prints on your GitHub or Linear webhook config. Webhook signatures are HMAC-SHA256 verified against the secret you provided at setup.
